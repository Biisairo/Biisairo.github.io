Mandatory

Project overview

  • virtual machine 의 작동에 대한 설명
  • 해당 os 를 선택한 이유
  • Centos 와 Debian 의 차이
  • apt 와 aptitude 의 차이
  • AppArmor 이 무엇인지
  • 스크립트가 잘 올라오고 있는지

Simple setup

  • 가상머신에 그래픽환경이 없다는걸 확인시켜준다
  • root 유저가 아닌 다른 유저에 로그인
  • 암호를 잘 확인할것
  • ufw 가 시작과 함께 작동하는지
  • ssh 가 시작과 함께 작동하는지

[[42/born2beroot/eval 3.png]]

User

  • 유저가 sudo 와 user42 그룹에 있는지 확인
  • 암호 룰 확인
    • 새로운 유저를 만든다. 암호를 설정하되 룰을 따라 만든다. 어떻게 룰을 설정했는지를 알려준다. 이때는 두개의 파일을 수정해야할 것이다.
    • 새로운 유저를 evaluating 이라는 그룹을 만들어서 안에 넣는다. 해당 유저가 evaluating 그룹에 들어간걸 확인한다.
    • 암호 룰의 이점, 장단점을 설명하라

Hostname and partitions

  • hostname 이 주어진대로 잘 되있는지 확인하라
  • hostname 을 바꾸고 재시작해라, 재시작하면 이름이 바뀌어있어야한다
  • 다시 원래의 이름으로 바꿔라
  • partition 을 보는 방법을 물어봐라
  • partition 은 과제에 주어진대로여야한다, 이때 보너스를 한 경우 보너스를 보아라
  • LVM 의 작동방법을 물어보아라

[[42/born2beroot/eval 4.png]]

Sudo

  • sudo 가 잘 설치되어있는지 확인해라
  • 새로운 유저를 sudo 그룹에 넣어라
  • 예시를 활용해 sudo 의 중요성과 운영방식을 설명하라
  • 과제에서 룰을 어떻게 적용했는지를 확인해주어라
  • /var/log/sudo 를 확인시켜줘라. 안의 내용을 확인하고 sudo 커맨드를 사용한 기록을 보여준다
  • sudo 커맨드를 활성화 한 다음 로그를 보여줘라

Ufw

  • ufw 가 잘 설치되어있는지 확인해라
  • 제대로 작동하는지 확인해라
  • ufw가 무엇이며 사용하는 이유에 대해 설명해라
  • ufw 에서 활성화 된 룰을 보여줘라. 룰은 4242 port 에 대한 룰이 있어야한다
  • 8080 port 를 다시 열고 활성화 된 룰에 추가가 되었는지 확인한다
  • 그리고 다시 해당 규칙을 삭제한다

SSH

  • ssh 가 잘 설치되어있는지 확인해라
  • 제대로 작동하는지 확인해라
  • ssh가 무엇이며 사용하는 이유에 대해 설명해라
  • ssh 는 4242 port 를 통해서만 인증되어야한다
  • 새로만든 user 로 로그인 하기 위해 ssh 를 사용하도록 한다
  • root 유저는 ssh 를 사용해서는 안된다

[[42/born2beroot/eval 5.png]]

Script monitoring

  • 스크립트가 어떻게 작동하고있는지 설명한다
  • cron 이 무엇인지 설명한다
  • 서버가 시작된 뒤 10분마다 자동으로 실행되도록 한 방법에 대해 설명한다
  • 설명이 끝난 뒤 스크립트를 매 1분(30초)마다 실행되도록 바꾼다
  • 결과가 동적으로 바뀌는지 확인한 뒤, 스크립트의 수정 없이 실행을 중지시킨다
  • 서버를 재시작하여 바뀐것을 확인한다
  • 재시작하면 스크립트가 원래 자리에 제대로 있는지, 권한이 변경되지 않았는지, 수정되지 않았는지 확인한다

Bonus

  • 파티션이 설정되어 있으면 2점
  • 과제가 요구하는 것 만으로 워드프레스 서비스를 설정했다면 2점
  • 자유 선택 서비스 1점
  • 각 서비스가 제대로 작동하는지 확인
  • 자유 선택 서비스가 왜 유용하다고 생각하는지에 대해 설명
  • NGINX 와 Apache2 는 금지
